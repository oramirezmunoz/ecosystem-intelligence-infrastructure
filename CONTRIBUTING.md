# Contributing to Ecosystem Intelligence Infrastructure

Thank you for being here. EII is an open project — built on the belief 
that solving information fragmentation in innovation ecosystems requires 
collective knowledge, not a single point of view.

Every contribution, large or small, moves this forward.

---

## What We Are Building Together

EII is an open framework — a growing body of knowledge, standards, 
playbooks, and tools that any ecosystem builder, policy maker, or 
practitioner can use to design and implement information infrastructure 
in their context.

It is not a finished product. It is a living document. The more 
diverse the contributors — in geography, sector, and experience — 
the more useful it becomes for everyone.

---

## Ways to Contribute

You do not need to be a developer to contribute. Most of what this 
project needs is knowledge and experience.

**Share implementation experience**
Have you built something that resembles EII in your ecosystem — even 
partially? Share what worked, what didn't, and what you learned. 
Real experience is the most valuable thing this project can receive.

**Improve the framework**
If something in the framework is wrong, incomplete, or could be 
expressed more clearly — open an issue and tell us why. Rigorous 
disagreement makes the framework stronger.

**Write a playbook**
The playbooks section needs context-specific implementation guides. 
If you have deep experience in a specific type of ecosystem — a city, 
an industry cluster, a national system — consider contributing a 
playbook for that context.

**Translate**
EII is a global problem. If you can translate any part of this 
framework into another language, that contribution directly extends 
its reach.

**Share examples and case studies**
Know of an ecosystem that has done something relevant — even 
imperfectly? Document it and contribute it. Partial examples are 
as valuable as complete ones.

**Improve tools and templates**
The tools and templates section needs practical artifacts that 
practitioners can pick up and use immediately. If you can design 
or refine these, your contribution has direct impact.

## How to Contribute

### Option 1 — Open an Issue (easiest)
If you have a suggestion, a correction, a question, or want to 
flag something missing:

1. Go to the **Issues** tab in this repository
2. Click **"New issue"**
3. Describe what you want to add, change, or discuss
4. Submit — we will respond and take it from there

This is the right starting point if you are new to GitHub or 
just want to share a thought without writing a full document.

### Option 2 — Submit a Pull Request (for direct contributions)
If you want to contribute content directly:

1. Fork this repository
2. Create a new branch for your contribution
3. Make your changes
4. Submit a pull request with a clear description of what you 
   added or changed and why

If you are not familiar with this process, start with an Issue 
instead and we can work through it together.

## Contribution Principles

**Quality over quantity**
A single well-reasoned insight is more valuable than a long 
document that hasn't been thought through.

**Plain language**
Write as if you are explaining to a smart practitioner who is 
new to the concept. Avoid jargon where plain language works.

**Cite your experience**
When you make a claim based on experience, say so. When you 
are speculating, say so. The distinction matters.

**Respectful disagreement**
The framework will be wrong about things. Push back when you 
disagree — but explain why, and propose something better.
